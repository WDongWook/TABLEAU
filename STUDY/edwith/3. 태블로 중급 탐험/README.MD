# 3. 태블로 중급 탐험


## Ep. 11 대시보드 만들기

- 대시보드 : 복수개의 워크시트로 구성되고, 시트 간 상호작용이 발생한다. 데이터를 발견하고 탐색한 후 그 과정에서 인사이트를 구하는 과정이다.
### 대시보드의 3가지 사항
1. 누가 볼 것인지 결정한다. -> 대시보드의 경우는 자기만족을 위해 구현하는 것이 아닌 다른 사람들이 보고 이해를 돕기 위해 만드는 것이다.
2. 보는 사람이 어떤 기기로 보는 것인지 중요하다. -> 보는 사람이 어느 장소에서 어떤 기기를 가지고 보는 것인지도 중요한 요소 중에 하나이다.
3. 보는 사람의 시선을 바로 잡을 수 있게 하는 것도 중요하다. -> 사람들의 시선은 주로 좌측 상단부터 내려오므로 중요하고 꼭 표현하고 싶은 것들이 있으면 좌측 상단에 두어 사람들의 시선을 사로 잡을 수 있다.


## Ep.12 스토리 만들기

- 스토리 기능을 사용할 때는 스토리 포인트의 순서에 따라 작성할 수 있다.

### 데이터 스토리 방식

1) 시간의 순서에 따른 구성
2) 요약에서 세부적인 정보로 드릴 다운
3) 관심사를 확대
4) A vs B로 비교 대조

### 스토리텔링을 잘 하기위한 5가지 사항

1) 누구를 위한 스토리인지, 청중이 관심이 있는지
2) 스토리포인트간 플로우를 잘 기획했는가
3) 데이터 시각화와 스토리 포인트가 일치하는가
4) 청중이 중간에 이탈하지 않고 스토리가 잘 전개되고 있는가
5) 스토리를 불러오는데 로드 없이 구성되어 있는가


## Ep.13 퀵 테이블 계산하기_1

- 누계 : 각각의 값을 누적해서 보는 것
- 차이 : 앞의 항목과 어느 정도 차이가 나는지를 구하는 것
- 구성비율 : 전체에서 각 멤버들의 비중을 확인할 때 활용하는 것
- 순위 : 각 멤버들의 측정값을 기준으로 순위를 나타낼 때 사용하는 것


## Ep.14 퀵 테이블 계산하기_2

- 비율차이 : 측정값들 사이의 성장률 or % 차이
- 전년대비성장률 : 같은 월 기준으로 이전 년도 대비 얼마 정도 성장했는지를 살펴보는데 활용
- 백분위수 : 전체에서 각 멤버들의 백분위를 표시. 상위, 하위로 각각 기준을 변경할 수 있음
- 이동평균 : 이전의 값부터 현재까지 값에 대한 평균을 낼 때 사용하며 주식 데이터에서 주로 이용
- YTD 총계 : 특정 시점을 기준으로 해당 연도부터 그 시점까지 총계를 의미
