# 4. 태블로 고급 맛보기

## Ep.21 효율적인 대시보드 만들기_1

### 효율적인 대시보드 만들기 1

1) 보는 사람이 누구인가?
2) 보는 사람이 보는 기기는?
3) 보는 사람이 사용하는데 느리다면?

- 대시보드는 자기 만족을 위해서 구현하는 것은 아니다.
- 따라서 보는 사람이 누구인지를 먼저 정의하는 것이 첫 번째로 우선이 되어야 한다.


- 태블로 영역에서 느리다면 살펴봐야 하는 곳은 필터 영역
-> 필터는 로딩 시간을 느리게 하기 때문

- 필터의 장점 : 분석할 때 데이터 양을 제한 두기에 편한 방법
- 필터의 단점 : 쿼리 성능에 영향을 준다.


### 성능을 높이기 위한 체크리스트

1) 마크의 수 줄이기
2) 필터 수 최소화
3) 대시보드 수 제한


## Ep.22 효율적인 대시보드 만들기_2

### 효율적인 대시보드 만들기 2

1) 보는 사람이 시선이 먼저 가는 곳 공략하기
2) 보는 사람이 집중하도록 Views & Colors 제한 두기
3) 보는 사람이 직접 참여하도록 유도하기

- 스윗 스팟(Sweet Spot)은 사람들이 특정한 화면을 보고 시선이 가장 먼저 그리고 가장 많이 가는 곳을 말한다.
- -> 만드는 이들(우리)가 공략할 곳은 바로 이 스윗 스팟

* 기본적으로 사람들의 시선이 가장 먼저 가는 곳은 좌측 상단 영역 *
